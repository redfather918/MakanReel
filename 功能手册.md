# MakanReel · 功能手册 (Phase 1)

> 对外产品名：**MakanReel**　|　内部代号：**NIHAO**
> 定位：为新加坡 180+ 门店餐饮品牌打造的「AI 短视频内容生成 Portal」
> 适用版本：Phase 1（极简上传 → AI 一键生成 → 预览/导出）

---

## 1. 一句话说明

门店员工在手机上上传 1~3 张产品图（或 1 段短视频），填一句备注，**点一下**即可得到一支竖屏爆款短视频 + 配套 Instagram Caption，**30 秒内**完成，可直接下载发布。

---

## 2. 核心使用闭环

```
┌─────────────┐    ┌──────────────┐    ┌──────────────┐    ┌─────────────┐
│ ① 极简上传   │ →  │ ② AI 预处理   │ →  │ ③ 模板合成    │ →  │ ④ 预览导出   │
│ 1~3图/1视频  │    │ 9:16智能裁切  │    │ 选风格+花字   │    │ 看片/下载/  │
│ + 备注       │    │ 画质修复      │    │ +BGM+文案    │    │ 复制文案     │
└─────────────┘    └──────────────┘    └──────────────┘    └─────────────┘
```

前端三态切换：上传页 → 生成中（进度轮询）→ 结果页。

## 3. 功能模块详解

### 3.1 📸 极简上传（Uploader）

| 能力 | 说明 |
|------|------|
| 素材数量 | 最多 **3 张图片** 或 **1 段短视频**（二选一，图片优先） |
| 图片格式 | JPEG / PNG / **HEIC / HEIF**（iPhone 实拍直传） |
| 视频格式 | MP4 / MOV / M4V |
| 大小限制 | 单张图片 ≤ 20 MB；单段视频 ≤ 50 MB、时长 ≤ 10 s |
| 交互方式 | 点击选择 / 拖拽上传 / 移动端直接调用后置摄像头（`capture`） |
| 预览 | 上传后即时显示 9:16 缩略图，可逐张删除重选 |
| 备注栏 | 可选填「产品 / 活动备注」，如「芒果啵啵 买一送一」，用于文案与选模板 |

> 备注不参与裁切，仅作为文案生成的输入与模板匹配的线索。

---

### 3.2 🤖 AI 智能预处理（Preprocessor）

对每张原图自动执行，用户无感知：

| 步骤 | 技术实现 | 作用 |
|------|----------|------|
| 主体定位 | OpenCV 谱残差显著性（Spectral Residual）计算视觉重心 | 找到产品/人脸中心，避免裁切切掉主体 |
| 人脸增强 | Haar 级联人脸检测（OpenCV 4.x；5.0 自动降级为纯显著性） | 有人物时优先对齐人脸 |
| 9:16 智能裁切 | 以主体为中心裁出 1080×1920 竖屏 | 适配 Reels / TikTok / Shorts |
| 画质修复 | CLAHE 对比度增强 + 去噪 + 锐化 + 亮度/对比度均衡 | 低光/压缩图也能出片 |
| YOLO 增强（可选） | `ENABLE_YOLO=true` 时加载 YOLOv8 识别杯/瓶/人 | 主体识别更准（默认关，需额外依赖） |

> 图片数少于模板片段数时，系统自动**循环复用**图片填满成片时长，保证时序完整。

---

### 3.3 🎬 模板库（Templates）

8 套内置风格，均可一键套用或「换风格重生」。花字（hook_text）会在视频前 2~2.5 秒居中弹出后淡出。

| ID | 名称 | 时长 | 风格 | 花字 (Hook) | 转场 |
|----|------|------|------|-------------|------|
| `summer_cool` | 夏日酷炫 | 8s | 活力清凉 | MUST TRY! | fade |
| `zoom_in` | 高质感 Zoom-In | 6s | 精致特写 | FRESH & DELICIOUS | zoompan |
| `promo_flash` | 促销快闪 | 5s | 紧迫高能 | BUY 1 FREE 1! | wipeleft |
| `ken_burns` | 静态微动 | 7s | 电影氛围 | SO SHIOK! | fade |
| `fresh_pop` | 清新气泡 | 8s | 明亮年轻 | REFRESHING! | slideleft |
| `night_vibe` | 夜间氛围 | 7s | 霓虹暗调 | NIGHT MOOD | fade |
| `sweet_zoom` | 甜蜜聚焦 | 6s | 温暖诱人 | SWEET TOOTH! | circleopen |
| `energy_burst` | 能量爆发 | 8s | 节奏冲击 | LIMITED TIME! | wiperight |

不指定模板时，系统自动选一套（重生时随机切换不同风格）。

---

### 3.4 🎞️ 视频合成引擎（Renderer）

| 能力 | 说明 |
|------|------|
| 画面动效 | `zoom_in`（缓慢推近）、`ken_burns`（电影微动）、`flash`（快闪脉冲+缩放）、`slide`（横向滑动）四种可组合 |
| 花字烧录 | 模板 hook_text 以描边大字居中弹出（前 2~2.5s 后淡出） |
| 背景音乐 | 程序化合成 BGM（按风格匹配节奏/音色），自动混音进成片 |
| 转场 | fade / zoompan / wipeleft / slideleft / circleopen / wiperight 六类 |
| Logo 位 | 模板定义角标位置（top-right / bottom-right），预留品牌挂载点 |
| 输出规格 | **H.264 + AAC，1080×1920（9:16 竖屏）**，MP4 封装 |

---

### 3.5 ✍️ 文案生成（Caption Writer）

| 模式 | 触发条件 | 行为 |
|------|----------|------|
| LLM 模式 | 配置了 `LLM_API_KEY` | 调用 OpenAI 兼容接口（默认 DeepSeek），按「新加坡 F&B 本地化」风格生成 2~3 句英文 Caption + 5 个 #SG / #Singapore 话题标签 |
| 模板兜底 | 无密钥 / 调用失败 | 自动组合本地语气词（Shiok! / Confirm shiok! / Mai tu liao! / Power! 等）+ 备注 + 固定 Hashtag 矩阵，**保证永远能出文案** |

**默认 Hashtag 矩阵**：`#SGFoodie` `#SGBubbleTea` `#SingaporeEats` `#SGDeals` `#SGDrinks`

LLM 文案要求：2-3 个短句、1-2 个本地俚语、结尾带行动号召（CTA）。

---

### 3.6 📲 预览与导出（ResultView）

生成完成后结果页提供：

| 操作 | 说明 |
|------|------|
| ▶️ 视频预览 | 9:16 竖屏内联播放，移动端静音自动循环（autoplay/loop/playsInline 兼容） |
| 📥 下载视频 | 直接下载 `MP4` 到本机，命名 `job_id.mp4` |
| 📋 复制文案 | 一键复制「Caption + Hashtag 矩阵」到剪贴板，直接粘进 Instagram |
| 🔄 换风格重生 | 复用原素材，套用其它模板重新生成（不重新上传） |
| 🎲 随机风格 | 随机选一套不同风格重生 |
| 生成新视频 | 返回首页，清空当前任务 |

---

### 3.7 🏷️ 品牌角标自动挂载（Brand Logo · Phase 2.1）

系统会在每段成片自动叠加品牌 Logo 角标，无需门店员工任何操作（后台配置生效，主路径仍是 3 步闭环）。

| 项 | 说明 |
|----|------|
| 角标位置 | 由模板 `logo_position` 决定：**右上角（top-right）** / **右下角（bottom-right）** |
| 透明度 | 默认 **90%**（可调 `BRAND_LOGO_OPACITY`） |
| 尺寸 | 默认占视频宽度 **15%**（`BRAND_LOGO_SCALE`，保持原图比例） |
| 边距 | 距边缘 **4%**（`BRAND_LOGO_MARGIN`） |
| 默认 Logo | `backend/assets/logo.png`（已内置 MakanReel 字标，透明背景） |
| 关闭方式 | 设 `BRAND_LOGO_PATH=""` 即不叠加 |
| 后台可配 | `GET /api/brand` 查看配置；`POST /api/brand/logo` 上传/更新 Logo（覆盖 assets/logo.png） |

> 角标叠加在花字烧录之后、缩略图之前完成，对所有模板统一生效；与"30 秒极简闭环"无冲突。

## 4. 技术规格速查

| 项 | 值 |
|----|----|
| 输出分辨率 | 1080 × 1920（9:16 竖屏） |
| 视频编码 | H.264 (libx264) |
| 音频编码 | AAC |
| 容器 | MP4 |
| 单视频时长 | 5~8 s（随模板） |
| 素材上限 | 3 图 或 1 视频 |
| 接口风格 | REST + 轮询（无 WebSocket） |
| 前端 | Next.js 14（移动端优先、Tailwind） |
| 后端 | FastAPI（异步任务编排） |

## 5. API 接口一览

| 方法 | 路径 | 作用 |
|------|------|------|
| GET | `/api/health` | 健康检查 |
| GET | `/api/templates` | 获取 8 套模板列表 |
| POST | `/api/upload` | 上传素材（form: `files`, `note`）→ 返回 `job_id` |
| POST | `/api/jobs/{id}/generate` | 触发生成（body 可选 `template`） |
| GET | `/api/jobs/{id}` | 查询状态/进度/视频/文案（轮询用） |
| GET | `/api/jobs` | 任务列表 |
| POST | `/api/jobs/{id}/regenerate` | 换风格重生 |
| GET | `/api/assets/{id}/video` | 下载成片 MP4 |
| GET | `/api/assets/{id}/thumbnail` | 取缩略图 |

> 交互式文档：`http://localhost:8000/docs`（Swagger）

## 6. 配置项（backend/.env）

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `LLM_API_KEY` | 空 | 填后启用 LLM 文案；留空走模板兜底 |
| `LLM_API_BASE` | `https://api.deepseek.com/v1` | OpenAI 兼容接口地址（支持 DeepSeek/OpenAI/Gemini） |
| `LLM_MODEL` | `deepseek-chat` | 模型名 |
| `ENABLE_YOLO` | `false` | `true` 启用 YOLOv8 增强主体识别 |
| `CORS_ORIGINS` | `http://localhost:3000` | 前端白名单 |
| `STORAGE_DIR` | `storage` | 运行产物目录 |

## 7. 典型操作流程（给门店员工）

1. 手机打开 Portal，点「📸 上传」拍/选 1~3 张产品图。
2. 备注填活动，如「芒果啵啵 买一送一」。
3. 点「✨ AI 一键生成爆款视频」。
4. 等进度条到 100%，看预览。
5. 点「📋 复制文案」→ 粘进 Instagram 发布框；点「📥 下载视频」→ 上传 Reels。
6. 不满意风格？点「🔄 换个风格」或「🎲 随机风格」秒重生。
